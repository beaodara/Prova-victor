//diferença ts e js // Em Angular, um endpoint é uma rota definida na API ou serviço que a aplicação se comunica para obter ou enviar dados. O HttpClient é usado para realizar solicitações HTTP aos endpoints. Os serviços encapsulam a lógica de comunicação com os endpoints e retornam os dados para os componentes. Aqui está um exemplo de como você pode criar um formulário em Angular para enviar os dados para o servidor e um endpoint para receber esses dados: Criação do formulário no Angular: app.component.html: html Copy code
app.component.ts: typescript Copy code import { Component } from '@angular/core'; import { HttpClient } from '@angular/common/http'; @Component({ selector: 'app-root', templateUrl: './app.component.html', styleUrls: ['./app.component.css'] }) export class AppComponent { formData: any = {}; constructor(private http: HttpClient) {} submitForm() { this.http.post('http://seu-endpoint.com/dados', this.formData) .subscribe(response => { console.log(response); }); } } Criação do endpoint no servidor para receber os dados: Supondo que você esteja utilizando Node.js com Express, você pode criar um endpoint para receber os dados assim: javascript Copy code const express = require('express'); const app = express(); app.use(express.json()); app.post('/dados', (req, res) => { const dados = req.body; console.log(dados); // Aqui você pode manipular os dados recebidos e retornar uma resposta adequada res.sendStatus(200); }); app.listen(3000, () => { console.log('Servidor rodando na porta 3000'); }); Certifique-se de substituir http://seu-endpoint.com/dados no código Angular pelo URL correto do seu servidor. Essa é uma maneira básica de criar um formulário em Angular e enviar os dados para o servidor através de um endpoint. No servidor, você pode manipular os dados recebidos da maneira desejada e retornar uma resposta adequada.
